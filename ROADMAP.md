# 美约管家项目规划

本文件记录后续开发方向。实现前先确认需求是否仍然成立；实现后把结果同步到 `CONTENT_UPDATES.md`。

## 当前状态

- v1 是本机保存的美甲美容院预约记账工具，不包含账号、云同步、短信通知、在线支付或会员系统。
- 数据保存在 IndexedDB；Android / iOS 通过 Capacitor 使用同一套前端代码。
- 当前默认员工为小美、傻大春、小甜，默认无示例预约。
- 主要业务已经覆盖：排班、预约详情、员工管理、服务项目、账本、货币显示、收入隐藏、草稿缓存。

## 短期规划

- 优化首次安装体验：增加“清空本机数据/恢复初始化数据”的安全入口，并提供二次确认。
- 完善预约详情：支持快速选择常用客户电话，或从最近预约中复用客户信息。
- 强化排班视图：支持快速跳到当前时间、员工筛选、当天预约搜索。
- Android 体验验证：在真机上检查安装、数据保留、横向滚动、双指缩放、日期选择和草稿恢复。

## 中期规划

- 增加数据导出：按日期导出预约和收入 CSV / Excel。
- 增加本机备份与恢复：导出 JSON 备份文件，导入时不破坏现有数据。
- 增加简单统计：按周/月查看预约数、收入、员工小计和服务项目排行。
- 准备正式发布：Android release 签名、应用图标完善、隐私说明和版本号管理。

## 暂不做

- 暂不默认加入登录、云同步、后台管理或后端 API。
- 暂不加入在线支付、库存、会员卡、短信/微信通知。
- 暂不支持每笔预约独立币种；货币仍是全店统一显示设置。

## 验收默认要求

- 前端功能修改后运行 `npm run build` 和 `npm run cap:sync`。
- Android 打包运行 `npm run android:build:debug` 并确认 APK 输出。
- 预约相关修改至少验证新增、编辑、删除、刷新后保留、冲突提醒、不同日期隔离。
- 数据结构修改必须兼容旧 IndexedDB 数据，不能默认清空用户已有预约。
