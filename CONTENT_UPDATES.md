# 美约管家内容更新记录

本文件记录项目已经完成的重要内容变化，方便后续维护、打包和交接。

## 2026-05-19 当前版本整理

- 项目文档体系补齐：`README.md`、`AGENTS.md`、`CONTENT_UPDATES.md`、`ROADMAP.md`、`MOBILE_BUILD.md`。
- 默认初始化员工调整为 `小美`、`傻大春`、`小甜`。
- 新安装时不再自动生成示例预约，首次打开保持干净排班。
- 手机端预约详情保持两列紧凑布局，备注独占整行。
- 日历和“今天”判断使用设备当前日期；当天营业时间内显示当前时间行。
- 当选择日期是今天时，已过去时间格显示为灰色。
- 设置页新增收入显示模式：全部显示、部分隐藏：首页收入、全部隐藏：只读金额。
- 设置页服务项目默认收起，展开后可新增、编辑、删除。
- 新建预约未保存时，会按日期、员工、开始时间缓存草稿；重新打开同一时间格可恢复未完成内容。
- 全 App 原生选择菜单已统一为底部弹窗样式，覆盖预约详情、营业设置、服务项目时长。
- 设置页新增客户记忆开关；预约详情输入客户姓名或电话时，会从已保存预约中提示补全另一项，回车可选中提示，也可继续手动输入新客户信息。
- 日历弹窗支持点击外部空白区域关闭；Android 系统返回键会先关闭当前弹窗/面板，主界面再弹出平滑动画的“是否退出 App？”确认框。
- Android debug APK 可通过 `npm run android:build:debug` 打包。

## 更新记录规则

- 每次完成用户可见功能、默认数据、数据保存逻辑、打包流程、移动端布局或业务规则变更，都在本文件追加一条记录。
- 记录以日期分组，写清“用户能看到什么变化”和“开发维护需要注意什么”。
- 不记录临时构建产物、未采用的尝试或纯格式调整。
- 如果一次更新同时改变了规划，记得同步 `ROADMAP.md`。
